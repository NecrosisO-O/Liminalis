# Liminalis Design Index

This directory stores design-phase artifacts for `Liminalis`.

## Principles

- Keep design decisions separate from architecture and implementation.
- Prefer durable product framing over premature technical detail.
- Mark assumptions clearly when they are not yet confirmed.

## Documents

- `design/01-product-definition.md`: product vision, problem framing, design principles
- `design/02-scenarios-and-scope.md`: target users, use cases, scope tiers, non-goals
- `design/03-open-questions.md`: unresolved product questions that need confirmation

## Current Status

- Design phase has started
- The current documents are a first-pass draft meant to establish a shared baseline
